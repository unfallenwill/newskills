# PRD 提取模式

不同 PRD 格式的详细提取模式。

## 模式 1：结构化 PRD

常见于有正式流程的成熟团队。遵循可预测的模板。

**特征：**
- 清晰的章节标题（背景、目标、功能、接口设计等）
- 编号需求列表
- 数据模型和 API 规格的表格
- 显式的验收条件章节

**提取策略：**
1. 将章节直接映射到五个提取区域。
2. 将表格解析为实体定义。
3. 编号列表直接映射为功能。
4. 验收条件章节自包含。

**结构映射示例：**
```
"1. 背景与目标"     → 上下文（帮助确定功能优先级）
"2. 功能需求"       → 区域 1：功能
"3. 用户故事"       → 区域 2：用户故事与场景
"4. 数据模型"       → 区域 3：数据模型
"5. 接口设计"       → 区域 3：接口
"6. 非功能需求"     → 区域 4：约束条件
"7. 验收标准"       → 区域 5：验收条件
```

## 模式 2：自由文本 PRD

常见于初创团队或早期项目。以叙述性散文撰写。

**特征：**
- 用自然语言段落描述功能
- 没有标准章节结构
- 需求和理由混杂
- 可能包含原型图引用

**提取策略：**
1. 先通读全文，建立整体理解。
2. 标记每个描述系统行为的动词短语。
3. 按实体对动词分组（什么对什么操作）。
4. 从叙述流程中提取隐式功能。
5. 从描述的结果中推导验收条件。

**关键启发式规则：**
- "应该能..." / "should be able to..." → 功能 + 用户故事
- "当...时" / "when..." → 场景触发条件
- "不能超过" / "must not exceed" → 约束条件
- "需要支持" / "must support" → 功能需求
- "如果...则..." / "if...then..." → 业务规则 / 条件逻辑

## 模式 3：混合 PRD

结构化和自由文本的混合。实践中最常见。

**特征：**
- 部分章节遵循模板，其余为自由文本
- 可能包含内嵌原型图、流程图或表格
- 需求细节程度不一

**提取策略：**
1. 先处理结构化章节（它们提供骨架）。
2. 从自由文本章节填补空缺。
3. 在章节之间交叉验证一致性。
4. 自由文本章节通常包含结构化部分未捕获的隐式需求

## 模式 4：用户故事看板

以用户故事集合的形式组织的 PRD，带优先级标签。

**特征：**
- 故事采用"作为...我想要...以便..."格式
- 优先级标签（Must have、Should have、Could have、Won't have）
- 每个故事下列出验收条件
- 可能包含故事点或时间估算

**提取策略：**
1. 每个用户故事直接映射为一个功能。
2. 从故事文本中提取角色、操作和收益。
3. 验收条件已按故事列出。
4. 按主题/史诗分组故事以识别数据模型。
5. 从故事交互中推导接口。

## 常见 PRD 章节对照表

| 章节名称           | 英文名称                | 映射到               |
|---------------------|-------------------------|----------------------|
| 背景与目标          | Background & Goals      | 上下文               |
| 目标用户            | Target Users            | 用户角色             |
| 功能需求            | Functional Requirements | 功能                 |
| 用户故事            | User Stories            | 故事与场景           |
| 交互设计 / 原型     | Interaction / Prototype | 场景、UI 规格        |
| 数据模型            | Data Model              | 数据模型             |
| 接口设计 / API      | API Design              | 接口                 |
| 非功能需求          | Non-functional Reqs     | 约束条件             |
| 验收标准            | Acceptance Criteria     | 验收条件             |
| 里程碑 / 排期       | Milestones / Timeline   | 任务排序线索         |
| 依赖关系            | Dependencies            | 外部约束             |
