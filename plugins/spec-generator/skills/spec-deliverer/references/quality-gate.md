# 质量门禁：输出交付

## 产出物

最终 spec 文件（本地 Markdown 和/或飞书文档）

## 目标

将审查通过的规格文档以用户指定的格式和位置交付，确保用户可以直接使用或转交给 coding agent。

## 使用场景

| 消费者 | 消费什么 |
|--------|----------|
| 用户 | 最终交付物，用于审阅或转交 |
| Coding Agent | spec 文件，作为实现指令的直接输入 |

## 验收标准

### 文件前置条件

- [ ] `{workspace}/tasks.md` 存在且步骤 1-5 状态均为 completed
- [ ] `{workspace}/review-report.md` 存在且审查结论为 PASS 或 PASS_WITH_NOTES
- [ ] `{workspace}/{feature-name}-spec.md` 存在且可读

### 输出质量

- [ ] 文件已成功写入指定路径
- [ ] 文件内容可读（非空、非截断）
- [ ] 文件内容与 `{workspace}/{feature-name}-spec.md` 一致（无遗漏）
- [ ] 文件路径符合用户要求（如用户指定了路径）

### 飞书输出（如适用）

- [ ] 飞书文档已成功创建/更新
- [ ] 飞书文档 URL 已返回给用户
- [ ] 飞书文档内容完整

### 用户确认

- [ ] 用户确认输出位置和格式（检查点通过）
